stm32最小系统

电源模块：AMS1117，5V->3.3V
复位电路：低电平就能复位了，再并连一块滤波电容
下载电路：通常是使用usb转232

启动模式

  1. 主闪存：flash启动，实际位置 0x0800 0000
  2. 系统存储器，实际位置 0x1FFF F000
  3. 内置SRAM：内存启动，速度最快，实际位置：0x2000 0000 ，配置好像有点麻烦，要将NVIC异常表和偏移寄存器什么的映射到SRAM中，看不太懂，有空再看
    1，2方式启动时，将内存和系统存储器的数据映射到0x0000 0000上，代码区默认从0x0000 0000开始执行
    允许从SRAM启动是STM32的特别之处，具体哪里特殊忘了记了…

内嵌自举程序

通信串口
使用接口： USART1/2：使用内置8m的HSI
CAN和USB：需要外部存在8M…我写的什么字啊我自己都看不清
51只能232不能485

485相当于232的升级版，正电平为1，负电平为0，降低了电压，保护芯片
使用内置的ISP程序下载的时候，要设置boot的值让stm32从ISP程序在的地方启动，自动下载到flash，下载完后再将boot配置成flash启动

时钟树

也就是说，初始的时钟只有4个，通过不断的分频，锁相环(PLL)和倍频，制造出很多实际的电平给具体的电器使用

锁相环使用压控振荡器，同时使用参考稳定的基准频率，似乎是比较参考电平与实际产生电平的值，再将得到的误差信号负反馈给输入信号，直至稳定，所以叫锁相环…具体的实现还不是很懂，有空看看…

总线与时钟源

SYSCLK: 一看名字就是系统时钟了，时钟信号来源？，通过AHB分频因子控制是什么鬼，再看看吧…
AHB(Advanced High Performance Bus)-> HCLK(High-speed clock)
APB(Advanced Performance Bus)-> PCLK(Peripheral clock) 用来挂载外设的时钟

数值关系

这里的数值关系使用的都是推荐值…
PCLK2 = HCLK = SYSCLK = PLLCLK = 72M，72m是推荐值，最高好像是100多m，但是官方推荐说在72m时可以保持稳定
PCLK1 = HCLK/2 = 36M, 没错，1是36m

其他时钟

  3. ADC时钟，通过PCLK预分配，什么采样周期，什么不小于1us的忘了
  4. RTC，realtime clock，用来给系统计时,由HSE/128
